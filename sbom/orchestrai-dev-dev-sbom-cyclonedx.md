# OrchestrAI-Dev - SBOM CycloneDX

## Overview

This document provides a comprehensive Software Bill of Materials (SBOM) for the OrchestrAI-Dev project in CycloneDX format. CycloneDX is a full-stack SBOM standard designed for use in application security contexts and supply chain component analysis. This SBOM catalogs all direct and transitive dependencies used in the project, including their versions, licenses, and relationships.

The OrchestrAI-Dev project is a React-based web application built with TypeScript, Vite, and modern web development tools. It provides AI-powered development tools including code quality analysis, testing automation, compliance checking, and documentation generation capabilities.

## Implementation

### Dependency Analysis Methodology

The SBOM was generated by analyzing the following dependency management files:

1. **package.json** - Located at the project root, defines all direct dependencies and devDependencies
2. **package-lock.json** - Provides resolved versions and integrity hashes for all dependencies
3. **Source code imports** - Validated actual usage of dependencies throughout the codebase

### CycloneDX Format Structure

The CycloneDX SBOM follows the specification version 1.5 and includes:

- **Metadata**: Project information, timestamp, and tool details
- **Components**: All npm packages with complete metadata (name, version, PURL, licenses, hashes)
- **Dependencies**: Full dependency tree showing relationships between components
- **External References**: Links to documentation, repositories, and package registries
