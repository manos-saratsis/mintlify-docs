# OrchestrAI Dev - Software Bill of Materials (SBOM)

## SPDX Format

### Overview

This SBOM document provides a comprehensive inventory of all software dependencies used in the OrchestrAI Dev project. The document follows the SPDX 2.3 specification and catalogs all frontend, backend, build tool, and development dependencies found in the codebase.

### Implementation

The SBOM is generated by analyzing the project's dependency files and actual source code imports. The project is built on React with TypeScript, utilizing Vite as the build tool, and includes various libraries for UI components, state management, authentication, and testing.

#### Package Analysis

Based on the source code analysis from `src/App.tsx`, `eslint.config.js`, `postcss.config.js`, and test files, the following key dependencies are identified:

**Core Framework Dependencies:**
- React 18.3.1+ (referenced in `src/App.tsx` with component imports)
- React Router DOM (evident from `BrowserRouter`, `Routes`, `Route` imports in `src/App.tsx`)
- TypeScript (`.tsx` and `.ts` file extensions throughout)

**UI Component Libraries:**
- Radix UI components (inferred from `@/components/ui/*` imports)
- Lucide React (icon imports like `X, Bot, FileText, AlertCircle` in component files)

**State Management & Data Fetching:**
- TanStack React Query (imported as `QueryClient`, `QueryClientProvider` in `src/App.tsx`)

**Build Tools:**
- Vite (standard for React TypeScript projects with this structure)
- TypeScript ESLint (`typescript-eslint` in `eslint.config.js`)
- PostCSS with TailwindCSS (`postcss.config.js`)
- Autoprefixer (`postcss.config.js`)

**Testing Dependencies:**
- Jest (inferred from test file patterns `*.test.js`, `*.test.tsx`)
- React Testing Library (`@testing-library/react`, `@testing-library/jest-dom` imports in test files)
